# AI Image Generation Prompts for ElBruno.MarkItDotNet

This document contains detailed prompts for generating visual assets for the **ElBruno.MarkItDotNet** library. These prompts are optimized for DALL-E 3, Midjourney, and similar AI image generation tools.

---

## Project Context

- **Project:** ElBruno.MarkItDotNet — A .NET library converting file formats (PDF, HTML, DOCX, TXT, JSON, images) to clean Markdown
- **Inspired by:** Python's markitdown
- **Target Audience:** .NET developers, AI/ML engineers
- **Use Cases:** RAG pipelines, documentation workflows, AI ingestion
- **Visual Metaphor:** Documents/files → Markdown → AI

## Asset 1: NuGet Package Icon

### Specifications
- **Dimensions:** 1:1 square (128x128 or 256x256 pixels)
- **Use:** Package icon on nuget.org
- **Design Purpose:** Represent "file format conversion to Markdown" concept
- **Requirements:** Clean, simple, recognizable at small sizes, scalable

### Design Guidelines
- **Color Palette:** Cool blues, teals, or greens with white accents; avoid gradients that blur at small sizes
- **Style:** Minimalist, geometric, icon-like
- **Content:** Single strong visual metaphor: file/document → arrow → markdown symbol (#) or flowing text
- **Background:** Transparent or solid complementary color
- **Font/Text:** Minimal to none; icon must work without text

No text, no shadows.

- No photorealistic textures or photos

- No 3D effects or depth

## Usage Notes

### For DALL-E 3
- These prompts are optimized for DALL-E 3's natural language processing
- Specify dimensions in the prompt (e.g., "1080x1080 square")
- Ask for "transparent background" or "no background" for icons
- Request "flat design" for clarity and scalability

### For Midjourney
- Use the aspect ratio parameter: `--ar 1:1` for icons, `--ar 1080:1080` for cards, `--ar 16:9` for headers
- Add quality boosters: `--q 2` or `--niji 6` for higher quality
- Use style modifiers if desired: `--style raw` or `--niji` for anime-adjacent styles
- Iterate with `/imagine` and refine based on outputs

### For Other Tools
- Adapt the prompts to match your tool's capabilities
- Test each prompt variant and iterate
- Consider generating 2-3 versions of each prompt for variety
- Use negative prompts as exclusion guidelines

### Best Practices
1. **Iterate:** Generate multiple versions and select the best
2. **Resize:** Ensure final images are properly scaled for their intended use
3. **Overlay:** For social cards and headers, test text overlays before finalizing
4. **Accessibility:** Ensure high contrast for readability and color-blind audiences
5. **Consistency:** Select variations that maintain a cohesive visual brand identity
